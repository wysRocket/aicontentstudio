## Components

- Authentication: user access and session management
- Firestore: content metadata, prompts, and workspace data
- Rules: security constraints in `firestore.rules`

## Validation checklist

- Auth providers enabled correctly
- Firestore indexes created for key queries
- Rules tested before production rollout
